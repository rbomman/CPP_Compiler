Pseudo-Assembly Instruction Reference
=====================================

The compiler lowers IR into a compact pseudo-assembly language executed by `virtual_cpu.py`.

Storage And Addressing
----------------------

`ALLOC name, size`

- Allocates `size` contiguous slots for `name`

`MOV dest, src`

- Writes a scalar or pointer value into `dest`

`ADDR dest, name, stride, span`

- Produces a pointer to `name`
- `stride` is used by pointer arithmetic
- `span` is the size of the pointed value for bounds-checked dereference/copy

`INDEXADDR dest, base_ptr, index, stride, span`

- Produces a pointer to an indexed element within a larger allocation

`FIELDPTR dest, base_ptr, offset, stride, span`

- Produces a pointer to a field at a slot offset within an aggregate

Pointer Reads, Writes, And Copies
---------------------------------

`LOADPTR dest, ptr`

- Reads the value at `ptr`
- Fails if the read is out of bounds

`STOREPTR ptr, value`

- Writes `value` through `ptr`
- Fails if the write is out of bounds

`COPY dest_ptr, src_ptr, size`

- Copies `size` contiguous slots
- Used for whole-struct assignment and struct-by-value call/return paths
- Fails if either range is out of bounds

Pointer Arithmetic
------------------

`PTRADD dest, ptr, offset`
`PTRSUB dest, ptr, offset`

- Moves `ptr.addr` by `offset * ptr.stride`
- Bounds are enforced later on dereference or copy

Arithmetic And Logic
--------------------

- `ADD`
- `SUB`
- `MUL`
- `DIV`
- `MOD`
- `POW`
- `AND`
- `OR`

Comparison And Control Flow
---------------------------

- `CMP`
- `SETL`
- `SETG`
- `SETLE`
- `SETGE`
- `SETE`
- `SETNE`
- `LABEL`
- `JMP`
- `JE`
- `JL`

Calls
-----

`CALL target -> dest`
`CALL target, arg1, arg2, ... -> dest`

- Calls `target`
- Stores a scalar return value in `dest`
- Struct-returning functions receive a hidden return-buffer pointer as argument 0

`RET`

- Returns from the current function

`HALT`

- Stops execution

Notes
-----

- The VM is address-based, not just name-based
- Structs are copied with `COPY`
- Arrays of structs and `->` access are supported
- Runtime failures include instruction context in the reported error
